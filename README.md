# คู่มือการตั้งค่าระบบลงทะเบียนโครงการ Open House 2569

ระบบนี้เป็นเว็บแอปพลิเคชันลงทะเบียนเรียนรู้สู่สายงานบุคลากรทางการแพทย์และสาธารณสุข ครั้งที่ 2 ปีงบประมาณ พ.ศ. 2569 ซึ่งบันทึกข้อมูลการลงทะเบียนจริงลงใน Google Sheet ของคุณ

---

## 📂 ส่วนประกอบของโครงการ
1. **`index.html`**: ไฟล์ระบบลงทะเบียนหน้าบ้าน (Web Frontend) ออกแบบให้รองรับการเปิดใช้งานบนโทรศัพท์มือถือและคอมพิวเตอร์ ด้วยสไตล์ที่พรีเมียม (Tailwind CSS และ SweetAlert2)
2. **`GoogleAppsScript.js`**: ไฟล์โค้ดระบบหลังบ้าน (Backend API) สำหรับนำไปวางในตัวแก้ไข Apps Script ของ Google Sheets เพื่อจัดการเรื่องโควต้า ตรวจสอบรายชื่อซ้ำ และเขียน/แก้ไขแถวข้อมูล
3. **`satun_schools.json`**: ไฟล์ข้อมูลรายชื่อโรงเรียนทั้งหมดในจังหวัดสตูลที่จัดหมวดหมู่แยกตาม อำเภอ -> ตำบล -> โรงเรียน เพื่อเป็นแหล่งอ้างอิงข้อมูลดิบ

---

## 🛠️ ขั้นตอนการติดตั้งและเชื่อมต่อระบบกับ Google Sheet

โปรดทำตามขั้นตอนดังต่อไปนี้เพื่อเปิดใช้งานระบบร่วมกับ Google Sheets ของคุณ:

### ขั้นตอนที่ 1: เตรียมโค้ด Google Apps Script
1. เข้าไปที่ Google Sheet ของโครงการ: [https://docs.google.com/spreadsheets/d/1Np7AeQonS5gDmFYF5pdssZ12FC_r3VTjBD55fdrw_OY/edit](https://docs.google.com/spreadsheets/d/1Np7AeQonS5gDmFYF5pdssZ12FC_r3VTjBD55fdrw_OY/edit)
2. ที่เมนูด้านบน คลิกที่ **ส่วนขยาย (Extensions)** > เลือก **Apps Script**
3. ลบโค้ดเริ่มต้นที่ระบบสร้างให้ในหน้าต่างแก้ไขออกทั้งหมด
4. เปิดไฟล์ `GoogleAppsScript.js` ในเครื่องคอมพิวเตอร์ของคุณด้วยโปรแกรมแก้ไขข้อความ (เช่น Notepad หรือ VS Code) จากนั้นคัดลอกโค้ดทั้งหมด
5. นำโค้ดที่คัดลอกมาวางในช่องแก้ไขของ Apps Script
6. กดปุ่ม **บันทึกโครงการ (Save Project)** (ไอคอนแผ่นดิสก์) ที่อยู่ด้านบนของช่องพิมพ์โค้ด

### ขั้นตอนที่ 2: เริ่มการทำให้ใช้งานได้ในฐานะเว็บแอป (Deploy Web App)
1. คลิกที่ปุ่ม **การทำให้ใช้งานได้ (Deploy)** ที่อยู่มุมขวาบน > เลือก **การทำให้ใช้งานได้ใหม่ (New deployment)**
2. ในหน้าต่างที่แสดงขึ้น ให้กดที่ไอคอนเฟือง (เลือกประเภท) > เลือก **เว็บแอป (Web app)**
3. กำหนดค่าต่างๆ ดังต่อไปนี้:
   - **คำอธิบาย (Description)**: ระบบลงทะเบียน Open House 2569
   - **เรียกใช้ในฐานะ (Execute as)**: ฉัน (อีเมลของคุณ @gmail.com)
   - **ผู้มีสิทธิ์เข้าถึง (Who has access)**: ทุกคน (Anyone) *(สำคัญมาก: ห้ามเลือกเป็น 'ผู้ใช้ที่มีบัญชี Google' หรือ 'เฉพาะฉัน' เพื่อให้บุคคลภายนอกกรอกข้อมูลได้โดยไม่ต้องผ่านการขอล็อกอิน)*
4. คลิกปุ่ม **การทำให้ใช้งานได้ (Deploy)**
5. ระบบจะแสดงหน้าต่างให้สิทธิ์ (Authorize access) ให้คลิก **ให้สิทธิ์เข้าถึง (Authorize access)** และล็อกอินเข้าบัญชี Google ของคุณ
   - *หมายเหตุ: หากหน้าต่างแจ้งว่า App Not Verified ให้คลิกคำว่า "Advanced" หรือ "การตั้งค่าขั้นสูง" ด้านล่างซ้าย จากนั้นคลิก "Go to... (unsafe)" หรือ "ไปที่... (ไม่ปลอดภัย)" เพื่อยินยอมให้สิทธิ์*
6. เมื่อ Deploy สำเร็จ ระบบจะแสดง **URL ของเว็บแอป (Web app URL)** (ลิงก์จะขึ้นต้นด้วย `https://script.google.com/...`) ให้ทำการ **คัดลอก (Copy)** URL นี้เก็บไว้

### ขั้นตอนที่ 3: อัปเดตลิงก์ในหน้าเว็บ Frontend
1. เปิดไฟล์ `index.html` ในโฟลเดอร์โครงการด้วยโปรแกรมแก้ไขข้อความ (เช่น VS Code, Notepad หรือ Sublime Text)
2. ค้นหาบรรทัดที่กำหนดค่าตัวแปร `API_URL` (ประมาณบรรทัดที่ 695 หรือค้นหาคำว่า `API_URL:`)
3. นำ URL ที่คัดลอกมาจากขั้นตอนการ Deploy ในขั้นก่อนหน้ามาวางใส่ในช่องเครื่องหมายอัญประกาศเดี่ยว ตัวอย่างเช่น:
   ```javascript
   API_URL: 'https://script.google.com/macros/s/AKfycbwZ...ของคุณ.../exec',
   ```
4. กดบันทึกไฟล์ (Save)

## 🧪 โหมดทดสอบระบบ (Mock Mode)

หากคุณ **เปิดไฟล์ `index.html` ดับเบิ้ลคลิกใช้งานทันทีโดยยังไม่ได้ใส่ `API_URL`** ในขั้นตอนที่ 3:
- ตัวระบบจะ **เปิดใช้งานโหมดจำลอง (Mock Mode)** ให้โดยอัตโนมัติ โดยจะจัดเก็บข้อมูลลงทะเบียนทั้งหมดไว้ในหน่วยความจำของบราวเซอร์คุณเอง (`localStorage`)
- ระบบจำลองจะกรอกข้อมูลนักเรียนสมมุติลงไป 300 กว่าคนให้โดยอัตโนมัติ เพื่อให้คุณเห็นสถิติแดชบอร์ดความจุที่นั่ง การตัดแถบสี และ Badger ความพร้อมรับ ("ว่าง", "ใกล้เต็ม", "เต็ม") ได้อย่างสวยงามทันที
- คุณสามารถทดสอบกรอกลงทะเบียนใหม่, ค้นหาประวัติด้วยเบอร์โทรศัพท์ และเข้าไปแก้ไขข้อมูลเปลี่ยนสถานที่ศึกษาดูงานได้เต็มระบบในแบบจำลองนี้ เพื่อตรวจเช็กการตอบรับของฟอร์มก่อนนำไปใช้จริงบน Google Sheet
